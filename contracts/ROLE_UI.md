# 역할 템플릿 — UI 스트림

## 1. 목표 (한 문장)
브리프가 지정한 **6단계 인수 데모를 처음부터 끝까지 클릭으로 시연할 수 있는**,
키보드만으로 완주 가능한(WCAG 2.2 AA) 화면을 만든다.

## 2. 소유 경계
- **소유**: `ui/`
- **읽기만**: `contracts/`, `ui/fixtures/`, `api/`, `logic/`, `core/`, `ocr/`, `eval/`, `pack/`
- **금지**: 위 어느 것도 수정하지 않는다. 계약 변경이 필요하면 **지휘자에게 보고**하고 멈춘다.

## 3. 입력 (전부 이미 존재함)
| 무엇 | 어디 |
|---|---|
| 응답 모양 계약 | `contracts/CONTRACTS.md` §2–§9 |
| 동결 상수 | `contracts/FROZEN_CONSTANTS.md` |
| **실제 파이프라인 출력 8종** | `ui/fixtures/*.json` |
| 모양 적합성 테스트 | `api/test_contract.py` (26개, 통과 중) |
| 좌표 변환 헬퍼 | `core/render.py` (`pdf_bbox_to_pixels`, `overlay_rects`) |

## 4. 제약 (위반 시 우승 불가 — 브리프 명시)
1. 🚫 화면 어디에도 **자격 판정을 표시하지 않는다.** `approve / deny / score / rank / priority` 금지.
   `READY_TO_REVIEW`는 "검토 준비됨"이지 "자격 있음"이 아니다. 오해할 문구를 쓰지 않는다.
2. 🚫 **보호속성 추론 금지.** 화면에 인종·장애·이민신분 관련 필드나 추론을 만들지 않는다.
3. 🚫 **자동 전송 금지.** 어떤 버튼도 패킷을 외부로 보내지 않는다.
4. ✅ **기권을 숨기지 않는다.** `abstentions[]`와 `review_reasons[]`는 반드시 화면에 나온다.
   접거나 작게 두지 않는다 — 이 제품의 핵심이다.
5. ✅ **모든 값 옆에 출처.** 값에는 문서 위 근거 상자가 붙고, 규칙 주장에는 인용·유효일자가 붙는다.
6. ✅ **키보드만으로 6단계 전부 완주 가능.** 마우스 없이 데모가 끝나야 한다.

## 5. 수용 기준 (주장하지 말고 측정할 것)
- [ ] 6단계 데모를 **처음부터 끝까지** 클릭으로 시연 가능
- [ ] **axe-core 위반 0건** — 스캔 결과를 파일로 남길 것
- [ ] 서버 없이(픽스처만으로) 화면이 뜬다
- [ ] FastAPI가 정적으로 서빙 가능 (빌드 산출물이 `ui/dist/`)
- [ ] 인터넷 없이 동작 (외부 폰트·CDN·이미지 금지)
- [ ] 계약에 없는 필드를 지어내 쓰지 않았다

## 6. 기술 선택과 그 이유
**바닐라 HTML/CSS/JS. 빌드 단계 없음.**

Lovable(AI 프론트 생성기)을 쓸 수 있었고 크레딧도 확보돼 있지만, 1차 구현에서는 쓰지 않는다:
- 우리 계약은 이미 동결돼 있고 픽스처도 실물이 있다. 생성기가 자기 데이터 모양을 만들면 재배선 비용이 든다.
- 빌드 단계·npm·외부 호스팅은 전부 **데모 중 죽을 지점**이다. 단일 프로세스·오프라인이 우리 교리다.
- WCAG 위반 0건까지 반복 수정하는 일은 우리 코드에서 직접 하는 편이 빠르다.

⚠️ 이는 지휘자의 **이전 계획 번복**이다(앞서 "Lovable이 반증을 무력화하는 열쇠"라고 했다).
번복 이유는 위 세 가지이며, **Lovable은 폐기가 아니라 보류**다 — 기능이 다 서고 시간이 남으면
같은 마크업에 시각적 다듬기 용도로 쓴다.

## 7. 보고 형식
무엇을 만들었는지, **axe-core 실측 수치**, 6단계 중 실제로 클릭 가능한 단계, 못 한 것과 그 이유.
숫자가 나쁘면 나쁜 대로 보고한다. 이 제품의 논지가 그것이다.
