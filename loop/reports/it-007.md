# it-007 — OCR work that cannot change the answer is skipped — ACCEPTED (operator-closed)
Change: after the text pass settles a document's expected fields, the OCR-words path runs
only if some field it could actually fill is still abstaining. Flag:
REALDOOR_OCR_SKIP_SATISFIED (default on; =0 restores the old behaviour bit for bit).

## The license
Performance only. The precondition is the injection gate's own abstention-only rule read
back: OCR words can only ever fill abstaining OCR-reachable fields, so when none exists
the work provably cannot change the output. Falsified over all 77 documents: 46 would
skip, and on every one the flag-on extraction JSON is byte-identical to flag-off.
Zero conflicts.

G1–G5 and G7 passed on four consecutive runs (flips: none, every time; pack 159/159,
