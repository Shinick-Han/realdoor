# it-015 — T28 (part 2) — skeleton wired to the model path; T25/T26 closed, T27 validated (gated), T21 leak-blocked

Wired the it-014 skeleton into the model mapper as CONTEXT and unified the caption
egress gate through the furniture test. **Deterministic path byte-identical (G5);
gate PASS G1–G7.** The wins are safety (T25/T26); T27 is solved *by reasoning* but
only with a frontier model, so it ships behind a default-off flag; T21 is left on
the table because its only route leaks.

## The design that shipped

- **Egress gate unified (T25).** `label_llm.model_mapper` now sends a caption only if it
  is furniture (colon or `STRUCTURAL_VOCAB`) — `skeleton.is_furniture_text`, context-free,
  so it holds on api/upload too. A person's name is not furniture, so it can no longer
  leave. When the extractor sets a page context, the egress set is the page's furniture ∪
  (opt-in) position-qualified captions, and the masked skeleton rides the user message.
- **T26.** A timeout is no longer memoized (was poisoning the caption as `unknown` for the
  run); the memo key now also carries a context digest, so one skeleton never answers for
  another document.
- **Flags (model-path only, inert on every deterministic run):** `REALDOOR_SKELETON_CONTEXT`
  (**default OFF**, see T27), `REALDOOR_SKELETON_POSITION` (**default OFF**, see T21).

## T27 — abstains by reasoning, but only with a frontier model

`measure_confirm_set --config with_mapper`, skeleton context ON:

| model | correct | wrong | ca_dlse hourly_rate |
|---|---|---|---|
| gpt-4o-mini | 26 | **3** | 19.55 (wrong) + 2 NEW `person_name` wrongs |
| **gpt-4o** | 26 | **0** | **abstain** |

`gpt-4o` reads the masked multi-row piece-rate table and declines — **T27 closed by
reasoning, no hand-coded refusal**, 0 wrong. `gpt-4o-mini` (the default model) reasons
*unsafely*: it nominates `person_name` on two ambiguous captions the confirm layer cannot
type-check away (`Address`, `are not available`) — the it-011 overconfidence hazard,
realised. So the context is safe only with a frontier model. Silently switching the
default model is what the operator asked not to happen, so context ships **default OFF**;
the validated opt-in is `REALDOOR_SKELETON_CONTEXT=1 REALDOOR_LABEL_MODEL=gpt-4o`.
**Cost:** the gpt-4o confirm run (14 docs) was ~$0.72 (~$0.05/doc, ~1000-token prompts);
gpt-4o-mini ~$0.04. Per-page context sends the skeleton per caption, so cost scales with
captions × pages.
