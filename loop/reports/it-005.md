## The guard, which is the deliverable
hi_ags prints THREE stub instances with three different value sets and nothing
printed selects one. Within a page: any band label printed more than once refuses the
page whole — measured live: page 2's merged dual-instance stream repeats TOTAL TAXES,
TOTAL DEDUCTIONS and NET PAY and refuses exactly there. Across pages: candidates that
disagree on any field withdraw everything (core/total_band.reconcile, pinned by
test) — found.setdefault's page order is never what decides between stub instances.
Said honestly, per design blunt-review 5: page 1 emits because page 2 refuses;
nothing printed *selects* the truth instance — the guard guarantees no conflicting
instance can ship a value, which is the deliverable.

## The license: printed total closed by an arithmetic identity
The four label strings are anchor-only (the shredded._ROW_LABELS precedent),
whole-cell exact match after normalize_label with spaces stripped (the engine's
measured word-space loss; no character is invented). FED TAXABLE GROSS can never
match (whole-cell equality) — the design's named adjacency risk, refused
structurally. Cents-form is required, so the '2.328.01' separator-misread family
never parses. Certainty "low"; each note names the band identity and OCR provenance.
