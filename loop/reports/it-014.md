# it-014 — T28 (part 1) — the privacy boundary, built and proven leak-free

Built `core/skeleton.py`: a position-aware, **leak-proof** page skeleton that
replaces every value with a typed placeholder while keeping the page's structure.
It is the fix for open defect **T25** (`assert_no_values` passes a person's name).
**No extraction behaviour changed** — the module is imported by nothing in the
extraction path; with `REALDOOR_SKELETON` unset it is not even built.

## The redaction rule (how a value is identified)

A run is **kept** only when it is confidently furniture: it ends in a colon (a
caption), or its whole `normalize_label` text is in `STRUCTURAL_VOCAB` (the
extractor's own `_WIDE_LABELS` + column-reader axis words + a general pay-stub
lexicon of headers and earnings/deduction row names). **Everything else is
redacted.** This is why it beats both shape and position: an employer name on a
34pt letterhead (`SMITH AND COMPANY, INC.`) is bold and in no value slot, so
position keeps it and "keep the bold structure" keeps it — but it is not structural
vocabulary, so this rule masks it. A person's name (`Terrence Boyd`, `Johnson,
Bob`) is caption-shaped, so shape keeps it — but it is not vocabulary either.
Position is used only to *type* a masked run (`<NAME>`/`<TEXT>`), never to keep it.

**Numeric half, proven by construction:** `STRUCTURAL_VOCAB` is filtered digit-free
and captions carry no digit, so the whole skeleton has **no digit** — no value with
a digit (amount, date, hours, id) can appear at all.

## Placeholder typing

`<MONEY>` (currency mark), `<DATE>` (m/d/y, masked `1/7/XX`, month-name), `<HRS>`
(number + hours unit), `<NUM>` (any other digit token), `<NAME>`/`<TEXT>`
(alphabetic value, whole run masked). Footnote markers and range/unit words survive
beside the placeholder: `36.83 *`→`<NUM> *`, `$19.37 **`→`<MONEY> **`,
`1/7/XX to 1/13/XX`→`<DATE> to <DATE>`, `24.00 hours`→`<HRS>`.

## The cost side (it-015 needs this)

Because the keep-list is a closed vocabulary, furniture it does not know is masked:
**printed instructions and footnote sentences** (the ca_dlse left-column callouts →
`<TEXT>`/`<NAME>`), and any **novel earnings-row label** not in the lexicon.
Deliberate — an instruction sentence is indistinguishable by shape from an entity
name, and over-masking a caption is cosmetic while under-masking a name is a leak.
it-015 should expect prose to arrive blanked and, if it wants callout text, must
find a signal that separates it from an employer name without leaning on shape.
