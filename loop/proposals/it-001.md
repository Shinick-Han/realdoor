# it-001 — T1 — the watermark filter may not delete the entire page

## 1. Target

Backlog **T1**, corpus `confirm`, doc `ca_dlse_paystub_hourly.pdf`. Fields:
`person_name`, `regular_hours`, `hourly_rate`, `gross_pay`, `net_pay` (all `abstain`).

## 2. The printed evidence (verbatim from `loop/worktmp/ca_dlse_paystub_hourly.words.json`)

The page is 1756.0 × 1176.0 pt — 2.2–2.9× the linear scale of the letter pages the pack
prints. Its text matrix draws every glyph at that scale: the title char `C` carries
matrix `[37.245, 0, 0, 36.6, 490.45, 1108.0]`, i.e. the page itself states a 36.6 pt
glyph. Measured over the whole page: **630 chars, 96 words, every char size in
27.3–48.6 pt, min 27.3** — so 100% of the text layer is at or above
`WATERMARK_MIN_SIZE = 20.0`, and `core.extract.read_words` returns **0 words** for a
page raw pdfplumber reads as 96. Body rows the filter deletes include (baseline · text):

- 742.0 · `Johnson, Bob XXX-XX-6789 18.00 regular 1/7/XXto 1/13/XX`
- 589.0 · `Regular 40.00 720.00 Federal W/H 60.45`
- 472.0 · `GROSS EARNINGS: 774.00`
- 396.0 · `NET EARNINGS: 560.71`

For contrast, a real watermark page (`pack hh-001_d01`): 521 chars, sizes 7.0–39.57,
**29.75%** at ≥20 pt (the diagonal `SYNTHETIC — NOT A REAL DOCUMENT` in Helvetica-Bold),
70.25% below — the filter leaves the body standing. That contrast — some large glyphs
over a smaller body — is the filter's premise, and this page prints no such contrast.

## 3. The license (exactly one of the five): **physical bound**

A watermark is physically an overlay on a body; the two coexist on the page by
definition. A size classification that, applied to a page, removes **every** char of its
text layer has refuted itself: it leaves nothing for the supposed watermark to overlay.
The page states the refuting fact directly — the printed distribution of its char sizes
has its minimum (27.3 pt) at or above the threshold — so the bound rests on what the
page prints, not on any new threshold. No constant is added, moved, or fitted:
`WATERMARK_MIN_SIZE` stays 20.0 and keeps doing exactly its job on every page that
prints the big-over-small contrast the filter was built for.

Alternatives weighed and refused:
- *Raise/remove the threshold*: admits the pack's 25.8–39.6 pt watermark glyphs into the
  body on the pack documents that print them — the named hazard below. Refused.
- *Page-proportional threshold* (scale 20 pt by page height): replaces one fitted
  constant with another fitted constant, and on this page would still delete the
  34.6–48.6 pt rows while keeping 27.3–29.0. Threshold-fitting; refused.
- *Reported size vs measured glyph-box height*: measured — unavailable. pdfminer reports
  `height == size` identically on every corpus doc probed (FontBBox fallback), so this
  reader has no independent glyph-box measurement to compare against. Refused.

## 4. The rule

In `core.extract.read_words`, the watermark size filter becomes conditional per page:
if the page has at least one text char **and every char reports
`size >= WATERMARK_MIN_SIZE`**, the filter is skipped for that page and the page is read
unfiltered; otherwise behaviour is unchanged, char for char. A page with no text layer
still returns `[]` (the scan case is untouched — the condition requires chars to exist).

Flag: **`REALDOOR_WATERMARK_SANITY`** — on by default; `=0` restores the unconditional
filter bit-identically (same convention as `REALDOOR_ARITHMETIC` / `REALDOOR_COLUMNS`).

## 5. The hazard, named

**A page whose entire text layer IS a genuine watermark** — a raster-bodied or blank
page carrying only a huge text stamp (`VOID`, `COPY`, `SYNTHETIC …`). The rule would
admit those giant words as body, and a stamp that prints digits (`VOID AFTER 90 DAYS`)
would put a number where the extractor can see it. What refuses the wrong value: the
rule changes only which words are *visible*, never what *licenses* a value — every
downstream emission still requires its own printed label, header cell, or closed
arithmetic identity, and watermark vocabulary prints none of those structures. The
residual risk (a stamp that accidentally prints label-plus-value shape) is exactly what
the falsification sweep must measure: the predicate runs the rule end-to-end on every
firing page and joins every emitted field against truth. Expected firing: **1 of 77**
(rasterized pages have zero text chars and cannot fire; every other corpus page probed
prints sub-20 pt body). Any conflict, or firing materially wider than expected, kills
the proposal before code is written.

The naive fix this proposal refuses — raising or removing `WATERMARK_MIN_SIZE` — would
re-admit the organizer watermark into the body on the pack documents and shred word
grouping (`MAILING AD DRESS`), the exact failure the filter exists to prevent.

## 6. Firing predicate

`loop/falsify/it-001.py` — read-only. `fires(doc)`: opens the PDF, fires on the first
page with ≥1 text char whose every char size is ≥ 20.0; on firing it also runs the
proposed rule in-process (monkeypatched reader, no repo writes) and reports every field
the full extractor would then emit. `conflicts(fired)`: joins each emitted field against
that document's truth (gold / manifest / truth-file shapes all handled); an emitted
field listed in `expect_absent`, or whose normalized value differs from truth, is a
conflict. Sanity: must fire on the target.

**No other field changes on any corpus.** Explicitly, on the target document itself:

- `person_name`, `hourly_rate` — **stay abstain**. Measured: with the page readable,
  both still abstain. They sit under the printed header row
  `EMPLOYEE  SOCIAL SECURITY NO.  PAY RATE  PAY PERIOD` (29 pt, non-bold), whose cells
  the header path does not recognise at this typography. That is a label/geometry cause,
  not reader loss — per design §8.1 these two return to the backlog as a new item with
  their own diagnosis rather than being patched inside this iteration.
- `pay_period_start`, `pay_period_end` — **stay abstain** (`masked`, `1/7/XX`): a flip
  there would mean the change invented a century and is a G7 anomaly by construction.
- `pay_date` — truth lists it absent; the extractor **stays abstaining**. Measured.

The three flipped values are the T1 gain this iteration claims: 3 of T1's 5 fields.
T1's remaining two fields become a successor backlog item (see §8 note).
