# it-014 тАФ T28 (part 1 of 2) тАФ the privacy boundary, built and proven

## 1. Target

Backlog **T28** тАФ widen the label model's EYES (a whole-page structural view)
without widening its HANDS (it still only nominates a field name; deterministic
geometry still reads every value). **This iteration builds and PROVES the boundary
ONLY. It changes no extraction behaviour.** No field on any corpus reads
differently; nothing consumes the skeleton yet. It is also the fix for **open
defect T25**: `assert_no_values` is a shape test that folds `$1,283.15` but passes
`Terrence Boyd`, because a name is caption-shaped. The redactor here is that fix,
proven leak-free before anything wider than a 48-char caption is ever sent.

The acceptance test is not a field count. It is: **byte-identity of all extraction
output (G5), and a leak-falsification that no truth value тАФ numeric OR personal тАФ
appears in any skeleton, over every corpus we own.**

## 2. The printed evidence (why position is not enough, and shape is not enough)

Empirically, on our corpora, the only non-numeric value fields are `person_name`
and `employer_name`. Where they sit in the text layer (probed on confirm+external):

- **Under/beside a caption** тАФ `Johnson, Bob` under `Employee:`; `ABC Staffing` in
  the beside-slot of `Employer:`. A pure *position* rule catches these.
- **As a free-floating letterhead/check title** тАФ `SMITH AND COMPANY, INC.` (34pt,
  alone), `Smith And Company, Inc.` (bold), `BONITA UNIFIED` (bold, twice),
  `Lansing Community College` (bold, ├Ч4), `AMIS SMITH` (bold header). These sit in
  **no value slot**, and they are **bold** тАФ so neither position nor "keep the bold
  structure" is safe. A title company name is a value, not furniture.

So shape leaks (a name is caption-shaped), and position leaks (a letterhead is
not in a slot). The only signal that separates `SMITH AND COMPANY, INC.` from
`GROSS EARNINGS` is that the latter is **structural vocabulary** and the former is
an open-class proper noun.

## 3. The license (exactly one of the five): **printed structure**

A run is kept only when it is **confidently furniture**: it ends in a colon (a
caption), or its whole normalized text is a member of a closed structural
vocabulary тАФ field labels, column headers, earnings/deduction row names
(`STRUCTURAL_VOCAB`, seeded from the extractor's own frozen `_WIDE_LABELS` and
column-reader axis words, extended with a general pay-stub lexicon; same KIND of
object as `LABEL_MAP`/`LABEL_SYNONYMS`). **Everything else is redacted.** An entity
name is never structural vocabulary, so it can never be kept, however bold or
caption-shaped. Position enters only to *type* a masked run (`<NAME>` vs `<TEXT>`),
never to decide keep-vs-redact. The house rule holds: a decision rests only on what
the page prints as structure; when a run cannot be classified with confidence it is
redacted тАФ over-masking a caption is cosmetic, under-masking a value is the one
leak this iteration forbids.

**One structural guarantee makes the numeric half unarguable:** the keep-list is
filtered to be **digit-free**, so the whole skeleton carries no digit. A digit-free
string cannot contain any value that has a digit тАФ every amount, date, hour count,
id. Only alphabetic values (names/orgs) then need the substring sweep.

## 4. The rules (a new, additive module: `core/skeleton.py`)

Nothing in the extraction path imports it. Reads `core.extract`'s reader/geometry
READ-ONLY (`read_words`, `group_lines`, `_split_runs`, `VALUE_Y_WINDOW`, ...).

- `is_furniture(run)`: colon-caption, or `normalize_label(text) тИИ STRUCTURAL_VOCAB`.
- `_redact_value_run(text)`: a purely alphabetic value тЖТ one `<NAME>`/`<TEXT>`
  (whole run masked, so no alphabetic token can survive as a stray "connective"); a
  digit-bearing run тЖТ token-by-token, each content token typed
  (`<MONEY>`/`<DATE>`/`<NUM>`/`<HRS>`) and only generic markers/units/range-words
  (`*`, `**`, `to`, `hours`) kept тАФ so `36.83 *`тЖТ`<NUM> *`, `$19.37 **`тЖТ`<MONEY> **`,
  `1/7/XX to 1/13/XX`тЖТ`<DATE> to <DATE>`. Footnote markers are the piece-rate cue T28
  names, and they survive beside the placeholder.
- `build_page_skeleton(words)` / `build_skeleton(path)`: reading order preserved,
  line breaks and left-to-right cell order faithful.
- `skeleton_enabled()` reads `REALDOOR_SKELETON` and gates only whether a skeleton
  is *built*. No reading depends on it; the extractor never calls this module, so
  G5 flag-off identity holds trivially.

## 5. The hazards, named (each with the defense that carries it)

- **A name that is caption-shaped** (`Terrence Boyd`, the T25 defect): masked by
  default-redaction тАФ it is not structural vocabulary, so it is never kept.
- **An employer name printed as a bold letterhead title**, in no value slot: same тАФ
  default-redaction, not vocabulary. Proven on `SMITH AND COMPANY`, `BONITA
  UNIFIED`, `Lansing Community College`, `AMIS SMITH`.
- **A numeric value surviving**: structurally impossible тАФ the skeleton is
  digit-free by construction (`STRUCTURAL_VOCAB` filtered of digit-bearing terms;
  captions carry no digit).
- **An empty redactor that "leaks nothing" trivially**: guarded by the inverse
  sanity тАФ known captions (`GROSS WAGES` on 4 docs, `EMPLOYEE'S NAME` on 8,
  `Rate/Hour`, `EMPLOYEE` on 90, ...) DO survive; 2386 furniture runs kept.
- **A value word colliding with a general caption** (`benefit_frequency = monthly`
  vs the kept caption `MONTHLY AMOUNT`): the *value slot* is masked to a placeholder;
  the only surviving `monthly` is inside a longer structural caption that prints on
  every benefit letter blank-or-filled. The predicate counts these 11 as *caption
  coincidences*, not leaks, and only when the value is a strict substring of a
  longer kept caption AND could not itself be kept as furniture.
- **Over-masking (the cost side)**: printed instructions, footnote sentences and
  novel (uncurated) row labels are masked, because they are indistinguishable by
  shape from entity names. Deliberate; reported in ┬зE for it-015.

## 6. Firing predicate

`loop/falsify/it-014.py` тАФ read-only. Builds every document's skeleton over all
seven corpora (pack, uploads, holdout, external, confirm, filled, scenarios), and
asserts **no truth value is a substring of its skeleton** (whitespace+case folded
and space-squeezed, both sides). filled/scenarios are read **DEV-ONLY** via
`build_field_state._dev_records`; no sealed hold-out is opened for its truth.
Inverse sanity asserts known captions survive. Written to
`loop/falsification/it-014.json`.
