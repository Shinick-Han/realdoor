## 2. The printed evidence (measured on the corpus text layers)

wa_dshs (blank, in the 77; and its filled twin) prints `EMPLOYEE’S NAME` with
the typographer's apostrophe U+2019 at baseline 585.6; `LABEL_SYNONYMS` holds
the ASCII form `EMPLOYEE'S NAME` (core/extract.py:407,470). `normalize_label`
(core/extract.py:533) folds whitespace and colons only, so the printed caption
is a total miss — while the frozen scorer's own `_fold`
(eval/score_extraction.py:176–180) folds `‘ ’ “ ”` to ASCII via `_QUOTE_MAP`
(lines 156–159) before comparing. The extractor refuses a string the scorer
would have accepted as the same string: an asymmetry, not a policy.

Glyph census over all 77 + the 3 filled dev docs (the enumeration the fix is
held to): U+2019 ×67 in 12 docs, U+201C ×31 in 6, U+201D ×31 in 6, U+2018 ×0.
No width variants of any quote glyph exist in the corpus. The only string
anywhere that folds onto a vocabulary key is wa_dshs's `EMPLOYEE’S NAME` (the
quote-carrying keys of the tables are exactly `EMPLOYEE'S NAME`, pay_stub +
employment_letter); every other occurrence is prose possessive or a caption
with no vocabulary twin (`Employer’s Signature`, `Workers’ Compensation`, …).

**Re-verified, and the backlog's "may move ZERO fields" was a lead, not a
fact:** the naive fold alone manufactures the corpus's would-be second WRONG.
On wa_dshs's row `EMPLOYEE’S NAME | EMPLOYER’S NAME` (two captions, one blank
cell each), folding makes the left caption a recognised label and the row
reader (`columns.column_value`, and `_side_by_side_value` on the same line)
reads the RIGHT caption as its value: measured, `person_name = "EMPLOYER’S
NAME"` at certainty low, on the blank (expect_absent → WRONG) and the filled
twin (truth `Terrence Boyd` → WRONG). The existing caption tests cannot refuse
it: no colon, and the row has only 2 caption cells (`MIN_HEADER_ROW_CELLS`=3).

## 3. The license (exactly one of the five): **printed label**

Two halves, one license, both reading only print already on the page:

(a) **The fold admits no new strings.** It maps the four glyph variants the
frozen scorer's `_QUOTE_MAP` already declares equivalent (`‘ ’ → '`, `“ ” → "`)
onto strings ALREADY in the closed tables — the same argument as
case-insensitivity in `_label_runs` path 2 ("this admits no new strings at
all"). The scorer is the project's frozen reading convention; the extractor
folding less than its own scorer is the asymmetry being repaired. Dashes are
deliberately NOT folded (the scorer folds `– — −`; labels distinguish hyphens —
`TAKE-HOME PAY` is a key, and orangeusd prints U+2010 ×14): fold set is the
four quote glyphs, enumerated, nothing else.

(b) **A parallel caption is a label, and a label is never a value.** When the
sole run beside a recognised label is caption-shaped (`_is_caption_cell`: no
digit, label-shaped) AND repeats the label's own final token (`EMPLOYEE'S
NAME` → `NAME` = `EMPLOYER'S NAME` → `NAME`), the page has printed the same
slot-naming construction twice on one row — the second is the next column's
caption, not a filled value. This restates the module's own rules ("a label is
never a value", "a caption we have no word for … act[s] as the boundary it
visually is") for the one caption shape those rules cannot yet see. The
refusal reads only printed tokens and is monotone: it turns a reading into an
abstention, never one value into another.

## 4. The rule (two seams, one flag)

Flag **`REALDOOR_QUOTE_FOLD`** — on by default; `=0` restores bit-identical
behaviour (both halves gated).

- `core/extract.py`: `_QUOTE_FOLD_TABLE = {‘→', ’→', “→", ”→"}`;
  `_quote_fold_enabled()`; `normalize_label` applies the translate before
  upper-casing when enabled. `_parallel_caption_refusal(label_run, run)` — the
  section-3(b) test, False when the flag is off.
- `core/extract.py` `_side_by_side_run`: after the existing gap test, refuse
  the run when `_parallel_caption_refusal(label_runs[index], run)`.
- `core/columns.py` `column_value`: refuse the chosen run (single-candidate
  and header-attributed branches) under the same helper. These are the two
  seams the measured wrong actually flows through; grounding measured the
  joint rule at zero emission diffs over every quote-carrying doc.

## 5. The hazards, named (each with the refusal that carries it)

- **Over-folding** (the backlog's named hazard): fold set is the scorer's four
  quote glyphs verbatim, enumerated in section 2 against the corpus census. No
  dashes, no U+2010, no NFKC, no width variants (none exist in the corpus).
- **The parallel-caption misread** (measured, section 2): the guard refuses it
  at both seams; wa_dshs blank + filled return to abstention.
- **Guard over-refusal**: a genuine value that is caption-shaped and ends with
  its label's final token (a person surnamed "Name") would abstain — refusal-
  biased, never a wrong; zero instances in 77 + filled dev (sweep is the record).
- **The guard is glyph-independent** (active on ASCII pages when the flag is
  on): a pack/uploads page printing `LABEL … CAPTION-TWIN` could newly abstain
  — a G3 regression if the old reading was correct. The P3 sweep runs the full
  extractor on all 77 + filled dev; zero emission diffs is the pass condition.
- **Recognition side-effects**: the folded label enters `label_words`/anchors
  and bounds columns on wa_dshs; measured zero emission diffs there beyond the
  guarded one.
- **Prose possessives** (`The employee’s name` in ext_les prose): a run must
  normalize EXACTLY to a table key; prose lines do not. Measured: ext_les
  emissions unchanged.
