# it-011 — T23 — one product, one reading path: the label ladder is the default

## 1. Target

`loop/backlog.json` T23. Not a single field on a single document — a **path
split**. `core/extract.py` exposes a three-tier label ladder (canonical
`LABEL_MAP` → `LABEL_SYNONYMS` → `core.label_llm.model_mapper`) but its own
public entry point `extract_document` defaults to `fallback_mapper=
synonym_mapper`, i.e. the **two**-tier path. A caller gets the third tier only
by asking for it by name.

The measured motivation is `../layout-generalisation-diagnosis.md` §5.1 and
§7.3: across 754 reachable expected fields, 22 hand-written phrases fitted on
the scenarios corpus scored **+242 in-sample and +0 / +0 / +0** on filled,
confirm and external. 84.1% of all 429 misses die at vocabulary. The closed
hand-written table *is* the overfitting mechanism. The model mapper is the
structural alternative already in the tree, measured at **55.9% → 76.5%
(+20.6pt, 0 wrong)** on the repository's own untouched wording hold-out.

**Correction to the iteration brief, established before any code was written.**
The brief states the model path "is enabled only on the upload route". That is
stale. `api/store.py::extract_one` has passed `tracking_layered_mapper()` since
commit `0099739` ("Measure it on paper we did not print"), an ancestor of
`e535463`, and its docstring says so. What is still split is (a) the library
default in `core/extract.py`, (b) `scripts/run_pipeline.py`, which calls
`extract_document` bare, and (c) `api/upload.py`'s prose, which still asserts
"팩 경로(`api/store.py`)는 표만 쓴 채로 둔다" — a claim its neighbour's code
contradicts. This iteration closes the default and corrects the prose.

## 2. The printed evidence

There is none, and that is the point of this iteration rather than a defect in
it. This proposal introduces **no new reading of any page**. It changes which
mapper a caller gets when it does not name one. Every geometric rule, every
window, every parse gate is untouched, and the value a model-named label points
at is located by exactly the code that located it yesterday.

The evidence that matters is therefore the transfer measurement, and it is §7's
job, not §2's.

## 3. The license: **not one of the five — no new value is read**

The five-license taxonomy governs *rules that read a value off a page*. This
proposal reads nothing. Stated precisely, because the distinction is the whole
safety argument:

- **The model sees one caption and nothing else.** No value, no coordinate, no
  page text, no household record. `core/label_llm.assert_no_values()` refuses
  to send a string that parses as money, a date, a bare decimal, an SSN shape
  or an email, and refuses anything over 48 characters as a sentence rather
  than a caption.
- **The model returns one name from a closed set**, collected at runtime from
  `core.extract.EXPECTED_FIELDS` — there is no hard-coded copy, so the list
  shown and the list validated against cannot drift. `unknown` is an explicit
  offered choice and the instruction prefers it. A name outside the set is
  discarded.
- **Deterministic code then confirms.** A named field's value must still sit in
  the column beneath the label inside `VALUE_Y_WINDOW`, aligned to
  `VALUE_X_TOLERANCE`, and must still `parse_value` as that field's type. This
  is the analogue of `api.route_llm.confirm()`'s anchor round-trip. **A wrong
  nomination therefore yields an abstention, not a wrong figure** — `SUPERVISOR`
  named `gross_pay` produces text that is not money and is discarded.
- **It runs last and may only fill blanks.** `_mapper_stages` splits the ladder
  into two ordered passes over the page, tables first; `found` carries across,
  so a label either table recognised can never be re-decided by the model. A
  judge with no key reproduces every table-resolved figure exactly.
- **It is capped at `certainty="low"` and tagged `MODEL_MAPPER_NOTE`**, counted
  in its own column by `scripts/measure_label_mapping.py`, so what is not
  reproducible offline is visible as such.

What this costs is **reproducibility for the fields it names**, not correctness.
That cost is why those fields are tagged and counted separately, and it is the
honest reason the split existed.

## 4. The rule (one seam, inside `core/extract.py`)

Flag **`REALDOOR_LABEL_LADDER`** — on by default; `=0` restores the two-tier
default bit-identically.

`extract_document`'s `fallback_mapper` parameter takes a **sentinel** default
rather than `synonym_mapper`. When the caller does not name a mapper, and only
then, the resolved default is `tracking_layered_mapper(doc_type)` with the flag
on and `synonym_mapper` with it off. An explicit `fallback_mapper=` argument —
which is what all six measurement harnesses pass — is honoured unchanged, and
`fallback_mapper=None` still turns the second pass off entirely.

`scripts/run_pipeline.py` keeps calling `extract_document` bare and thereby
inherits the ladder.

**Note the flag is not the model's on/off switch.** `core.label_llm.is_enabled()`
remains the model gate and remains off three independent ways (no
`OPENAI_API_KEY`, `REALDOOR_LABEL_LLM=0`, or under pytest without
`REALDOOR_LABEL_LLM=1`). With the ladder flag on and the model off,
`_mapper_stages` runs a second pass whose mapper returns `None` for every
string, which adds no field and moves no byte — already pinned by
`core/test_label_llm.py::test_layered_mapper_equals_synonym_mapper_when_model_is_off`.

## 5. The hazards, named

**H1 — the model reaches into territory the deterministic layer owns.** If
enabling the ladder globally moves a single pack byte, the design claim (the
canonical table catches every pack phrase first, so tier 3 is never reached) is
false. *Refusal:* `found` carries across passes and tier 3 is asked only for
labels both tables returned `None` for. *Proof obligation:* pack byte-identity
under flag-ON **and model-ON**, not merely under flag-off. This is a stop-and-
report condition, not a tuning target.

**H2 — a wrong nomination becomes a wrong value.** *Refusal:* the closed-set
check, the unchanged geometry, and `parse_value`. A wrong name for a typed field
yields an abstention. The residual is a wrong name for a field whose type is
permissive — the `FREE_TEXT_FIELDS`, chiefly `person_name` and `address`, where
any string parses. *This is the one real exposure and it is what the zero-wrong
bar across all six corpora is measuring.*

**H3 — a caption that is really a value leaves the process.** *Refusal:*
`assert_no_values`, whose `YYYY-M(M)` shape was added after `2026-06` was caught
leaving. Counted as `rejected_value_shaped`.

**H4 — a transient timeout permanently poisons a label.** `model_mapper` writes
`None` into `_MEMO` on timeout, so one slow call makes that caption unreadable
for the rest of the process, and re-running the same harness can score
differently. Observed live during this iteration's smoke test: `Team Member:`
returned `None` at the 6s default and the memo held it. *Refusal:* none in code
today. *Mitigation for measurement:* `REALDOOR_LABEL_TIMEOUT=30` on every
measured run, and timeouts reported as a stat. **Filed as a successor, not
fixed here** — fixing it means distinguishing "the model said unknown" from "we
never heard back", which is its own iteration.

**H5 — the cache serves model-fed extractions to a no-model run.**
*Refusal:* `api/store._label_model_key()` already puts
`is_enabled():MODEL` in `_cache_key`, and `core/label_llm.py` is in
`ENGINE_SOURCES`. Verified, not assumed — §7 reports the key under both states.

**H6 — cost and latency on every document.** *Refusal:* tier 3 is reached only
by labels both tables missed; `_MEMO` asks each distinct caption once per
process; the gateway caches across processes. Reported as call counts.

## 6. Firing predicate

`loop/falsify/it-011.py`: for each of the 77 manifest documents, the predicate
fires when the resolved default mapper differs from `synonym_mapper` **and** the
document presents at least one label run that both tables return `None` for —
i.e. a label that would actually reach tier 3. Read-only; it asks
`unmapped_labels(mapper=synonym_mapper)` and reports the strings, without
calling the model.

This predicate is deliberately an *upper bound* on where the change can act.

## 7. Predicted flips (exact)

**Pack: exactly zero.** Any pack flip is H1 and stops the iteration.

On the real-carrier corpora the honest prediction is a range, not a list,
because the model's answers are not knowable without asking it: the diagnosis
puts 361 of 429 misses at vocabulary, and the hold-out measured the model
recovering ~20.6pt of a 44-point gap. Predicting individual fields would be
predicting the model. The iteration therefore reports **observed** flips
field-by-field with the caption each read, and G7's "observed ⊆ predicted" is
recorded as **not satisfiable in its usual form** for the same registry reason
it-008 and it-010 filed: `baseline.field_state` does not enumerate the filled or
scenarios corpora at all.
