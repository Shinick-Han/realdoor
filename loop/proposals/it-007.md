# it-007 — PERF-1 — skip the OCR-words pass when no field it could touch is still abstaining

## 1. Target

Not a backlog abstention: a **performance** iteration, field yield 0 by design.
Measured on this machine, engine warm: `il_dol` (pure text, no images) ~0.1s;
`bonita` ~4.4s; `hi_ags` ~19.3s — the OCR-words path renders and recognises
embedded raster regions on every document that has them, whether or not anything
downstream could consume the result. On the deployed CPU these multiply 2–4x,
and a judge uploading their own PDF pays it. Target: documents whose OCR pass is
structurally dead work stop paying for it; every other document unchanged;
**zero flips anywhere** — the predicted flip set is EMPTY and any observed flip
is an automatic reject.

## 2. The structural evidence (read from the call sites), and the exact frontier

`ocr_by_page` is written once (`core/extract.py` page loop, under
`_ocr_words_enabled() and _arithmetic_enabled()`) and read in exactly four
places, all inside the `_arithmetic_enabled()` block:

1. the `verified.verify_page` loop — runs only `if wanted:`, where `wanted` is
   the blank subset of `EXPECTED_FIELDS[doc_type]` ∩ `verified.VERIFIABLE_FIELDS`
   ("blank" = missing from `found` or `certainty == "abstain"`), computed AFTER
   the text pass;
2. the `shredded.recover` loop — gated on `shredded_wanted` = blank ∩
   `("gross_pay", "net_pay")`;
3. the `total_band` block — gated on `band_wanted` (same set) before
   `any(ocr_by_page)` is evaluated;
4. `_with_ocr_provenance` — wraps only values returned by 1–3, assigned only
   where `_blank(name)` holds.

Nothing else sees an OCR word: `all_words`, `has_text_layer`,
`find_untrusted_text`, the abstention fill and staleness read printed words only
(it-003's licensing structure). So the **only fields any OCR consumer can ever
ask for** are `EXPECTED_FIELDS[doc_type] ∩ (VERIFIABLE_FIELDS ∪ {gross_pay,
net_pay})` — the numeric four. The exact precondition follows: **if, after the
text pass, none of those fields is blank, every consumer is unreachable and the
OCR pass may be skipped entirely.** If any is blank, collection must run exactly
as today.

**Why the wider phrasing "no expected field abstaining" is NOT the frontier, and
why bonita cannot be skipped at all.** bonita's text pass reads its five present
fields, but `hourly_rate` and `regular_hours` — absent on the page, truth-side
`expect_absent` — remain abstaining, and both are in `VERIFIABLE_FIELDS`. So
`wanted` is non-empty on bonita today, `verify_page` runs WITH the injected OCR
words, and it is structurally licensed to emit either field (factor columns of
an anchored run) or to attach an adjacency proposal to their abstention notes
(`_propose` reads the full injected word list). Skipping bonita would be
corpus-fitted, not output-identical by structure — its 4.4s stays, honestly.
Measured over all 77 (text pass only): the exact precondition fires on 46, the
narrower phrasing on 36; five firing documents carry embedded images
(`hh-002_d01`, `hh-005_d01`, `up_010_benefit_letter_sam_poe_scan`, `ext_nydol`,
`ca_dlse_paystub_hourly`) and stop paying render+recognition; the rest stop
paying only region discovery. `up_010` is the sharpest case: a full-page scan
whose document type (`benefit_letter`) expects NO numeric-four field, so its
whole-page OCR is dead work by type alone — vacuous satisfaction is exact, not
approximate, because `wanted`/`shredded_wanted`/`band_wanted` derive from the
same empty intersection.

## 3. The license: performance only — the license is the injection gate's own abstention-only precondition

No new acceptance rule, no threshold, no geometry. The skip fires exactly when
the it-003 injection gate's own design already guarantees the OCR words could
not be consumed. Output-identical by structure, not by tuning. **Hazard: the
precondition being approximate.** What was checked, exhaustively: every read of
`ocr_by_page` (the four sites above — grep confirms no fifth); that the three
consumer gates all derive from the same blank test over the same field set, so
"no blank numeric-four field after the text pass" empties all three (later
`_blank` re-evaluations can only see fields *fill*, never un-fill — `found`
entries are only added, never removed); that `_with_ocr_provenance` touches no
field outside those blank-gated assignments (no bbox, note or certainty of a
settled field can change); that abstaining fields OUTSIDE the numeric four
(dates, names, frequencies) are untouchable by every OCR consumer —
`VERIFIABLE_FIELDS` is the closed list `verify_page` speaks about, and
shredded/total_band restate `("gross_pay", "net_pay")`; and that deferring
collection (§4) cannot reorder output, because `region_ocr_words` is a pure
deterministic function of `(render_source, page, page_number, words)` and
`ocr_by_page` is rebuilt in the same page order (verified empirically on bonita:
eager and post-loop collection produce identical Word streams).

## 4. The rule (one mechanism, inside `core/extract.py`)

Flag **`REALDOOR_OCR_SKIP_SATISFIED`** (on by default; `=0` restores today's
conduct byte-for-byte). With it on, the page loop no longer collects OCR words
eagerly; `ocr_by_page` fills with empty lists. After the loop — inside the
`pdfplumber.open` block, after the text pass has settled every page and before
any OCR-words work — the precondition is evaluated: if any name in
`EXPECTED_FIELDS.get(doc_type, ())` that is in `verified.VERIFIABLE_FIELDS` or
in `("gross_pay", "net_pay")` is blank in `found`, `ocr_by_page` is rebuilt by
calling `ocr_words.region_ocr_words(render_source, page, page_number, words)`
over `zip(pdf.pages, words_by_page)` in page order — exactly the calls the
eager loop made, deferred. If none is blank: no discovery, no rendering, no
recognition, and `core.ocr_words` is never imported (same import discipline as
`REALDOOR_OCR_WORDS=0`). The `verified` import this check needs happens only
when `_arithmetic_enabled()` is on — the same condition under which the
arithmetic block imports it today. With the flag `=0` the eager per-page
collection runs in its original place, unchanged.

## 5. The hazards, named

- **An approximate precondition** — the killer. §3 names every read site
  checked; the P3 predicate does not trust the reading: it byte-compares the
  full extraction JSON, skip conduct vs committed conduct, on every firing
  document. Any byte difference on any document = conflict = reject.
- **The tempting over-wide rule** — "bonita reads 5/5, skip it" — refused in
  §2: 5/5 of the *truth's* fields is not 0 blank among the *consumers'* fields.
  The predicate's sanity leg pins the refusal: it must NOT fire on bonita,
  hi_ags, ou or lcc.
- **Deferral changing output on non-skipped documents** — refused by purity of
  `region_ocr_words` (own pypdfium2 handle per call; reads only the parsed
  `plumber_page.images` / `.height`; page order preserved), and pinned
  end-to-end by test (c).
- **A blank that appears after the text pass** — does not exist: `found` values
  are only ever added where `_blank` holds, never removed.
- **The cache masking the comparison** — every measurement clears
  `.cache/extractions` first (loop README footgun 1); `core/extract.py` is in
  `api/store.ENGINE_SOURCES`, so deployed caches invalidate on this edit.

## 6. Firing predicate

`loop/falsify/it-007.py` — read-only; replicates the committed text pass
verbatim (`read_words` + `extract_fields_from_page` with the deterministic and
synonym mappers, `found.setdefault` per page) to compute the precondition, then
for every satisfied document runs the full committed extractor twice — once as
shipped, once with the skip conduct emulated exactly (`REALDOOR_OCR_WORDS=0`
produces the identical all-empty `ocr_by_page` without importing the module) —
and byte-compares the two full-document JSON dumps. `fires(doc)` = the skip
would fire; `conflicts` lists every fired document whose dumps differ by one
byte or more. Unsatisfied documents take today's code path literally and are
not run twice. Sanity: must fire on `ca_dlse_paystub_hourly.pdf` (numeric four
all read from text; one embedded image whose render it saves) and must NOT
fire on bonita, ou, hi_ags or lcc. EXPECTED to fire widely (46 of 77 — the
pack alone is 24 text-complete documents): wide firing is wide savings; the
safety claim is the byte-identity join, not the firing count.
