## 3. The license (exactly one of the five): **printed structure**

The page prints what makes an option list an option list: **three or more
candidates for the same field, sitting on one band of type.** An assertion needs
one answer; a band offering three cannot be supplying it, whichever of them we
happen to recognise. So no run on such a band may anchor that field — it abstains
exactly as it did before any mapper was attached. This rests on printed geometry
(same-field multiplicity + baseline proximity, read through whatever mapper the
pass carries), never on a vocabulary of frequency words — the next form's options
are words nobody listed.

**Marks are out of scope, deliberately.** A ticked box or circled word still
refuses the whole band; a marked menu abstains where it could in principle be
read. That is the safe direction and it is honest about what a word reader sees:
the text layer carries all menu words equally and a vector ellipse is not legible.
Reading the marked option is a successor.

A second, smaller license (**printed punctuation**): a run with a colon *inside*
it is a caption and its own gloss — a page describing a slot, never a free-text
value. This is the terminal-colon rule of `_caption_refusal` read one notch
further, same scope (`FREE_TEXT_FIELDS`).

## 4. The rules (two seams, both in `core/extract.py`)

Flag **`REALDOOR_OPTION_BAND`** (on by default; `=0` bit-identical):

- `_on_one_band((baseline, size), …)`: two runs are on one band iff their
  baselines differ by less than the **smaller of their type sizes**. Derived, not
  fitted: the type size is the page's own statement of its line pitch, so seattle's
  staggered 4.56pt/8.04pt menu is one band and ca_dlse's 10.68pt/7.68pt table rows
  are not.
- `_option_menu_runs(lines, document_type, field_mapper)`: groups every
  `_label_runs` run by the field its mapper names, sorts by baseline, and marks
  every run in a band of `≥ MIN_OPTION_CELLS` (=3, the same threshold and reason
  as `MIN_HEADER_ROW_CELLS`). Computed once per page and threaded like
  `header_words`.
- `_scan_page`: a run whose `(field, id)` is in the menu set is skipped as an
  anchor. It stays a label everywhere else — bounds columns, is in `label_words`,
  shields what sits beneath it — so the rule only ever withholds.

Flag **`REALDOOR_GLOSS_COLON`** (on by default; `=0` bit-identical): one clause in
`_caption_refusal` refuses a `FREE_TEXT_FIELDS` candidate carrying a colon inside
it, alongside the existing terminal-colon clause.

## 5. The hazards, named (each with the refusal that carries it)

- **A real two-column layout** (a caption printed once over each of two column
  groups; `bonita`'s `GROSS EARN'S` twice): threshold is **three**, not two —
  `MIN_OPTION_CELLS = 3`, pinned by test.
- **A rate table read as a menu** (the ca_dlse hazard): `_on_one_band` refuses to
  merge rows a full line of their own type apart, pinned by the real numbers.
- **The rule eating a real single-value field**: it fires only on same-field
  multiplicity ≥ 3 on one band; measured, that is 3 confirm docs + 2 filled twins
  and nothing else in 77 + 157 dev.
- **A gloss colon swallowing a real value**: scope is `FREE_TEXT_FIELDS` only, and
  no name/address/frequency value is printed with an interior colon; typed fields
  are already refused by `parse_value` (`Advice Date: …` is not money).
- **The dropped third rule** (`ca_dlse` `hourly_rate`): a "column prints several
  different answers → abstain" refusal was written and **falsified by the test
  suite** — `test_extract_reading.py::TestRightAlignedColumnsAreMeasured` proves
  `ext_unc` prints the current-period figure and its YTD rival in one aligned
  column under `NET PAY` and reading the closer one is a deliberate capability.
  The corpus sweep did not catch that; the unit test did. Filed as **T27**.

## 6. Firing predicate

`loop/falsify/it-013.py` — read-only; extracts each document twice in-process,
`REALDOOR_OPTION_BAND`+`REALDOOR_GLOSS_COLON` off vs on, model ON, and fires iff
the emitted set differs. Because this rule *removes* readings, `conflicts` is
two-sided: (1) a flag-on emission truth contradicts, and — the one that matters —
(2) a flag-off value that **agreed with truth** and is now gone. Swept over the 77
manifest via `run_phase.py p3 --iteration 13 --run`, then `--dev-corpora` for
filled + scenarios; sealed sets never opened. Requires `REALDOOR_LABEL_LLM=1`:
with the model off the menu words are named by neither table and no menu forms.
