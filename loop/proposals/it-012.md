# it-012 — T18 — a value typed into a form widget is text the page prints

## 1. Target

The AcroForm blind spot. pdfplumber reads the page content stream; a value typed into
an interactive form field lives in a widget annotation and its glyphs live in that
annotation's appearance stream, so a correctly filled agency form reads BLANK.

Measured, not assumed: `testdata/filled/wa_dshs_14252_employment_verification_filled.pdf`
carries 17 filled text widgets (`EMPLOYEES NAME` = "Terrence Boyd", current-month gross
= "1,283.15") and `page.extract_text()` contains neither string; all 7 of its expected
fields abstain. Of the 11 real blank forms collected from public agencies, 7 are
AcroForms; Reno Housing's rental application has 500 fields.

Target fields: the `acroform` scenario stratum (8 sets, built expressly as this
iteration's acceptance corpus, currently reading 0) and the filled dev carrier.

## 2. The printed evidence (measured on the page)

Swept every corpus with pypdf. **12 of 264 documents carry an AcroForm:**

| corpus | docs | with AcroForm | with a FILLED text widget |
|---|---|---|---|
| pack 24 | 24 | **0** | 0 |
| uploads 26 / holdout 7 | 33 | 0 | 0 |
| external 6 | 6 | 2 (`ext_nydol`, `ext_va`) | **0 — blank forms** |
| confirm 14 | 14 | 1 (`wa_dshs` blank) | **0 — blank form** |
| filled dev | 3 | 1 | 1 (17 values) |
| scenarios | 182 | 8 (the `acroform` stratum) | 8 (11 values each) |

So on the 77-document manifest **the module engages on 3 documents and emits nothing on
all 3**: every manifest AcroForm is an unfilled blank. Pack carries no AcroForm at all —
the invariant "this is free on pack" is verified, not asserted.

Three further page facts, each measured:

* **Every filled text widget has an appearance stream** with the value baked in as a
  `Tj`. pdfplumber never descends into them (proven by the blank `extract_text()`), so
  there is no double-read today — but a *flattened* form would print the same glyphs
  into the content stream while keeping `/V`, and that is a real double-read.
* **`/Rect` is not the value's box.** `EMPLOYEES NAME` has a 210.7 × 21.5pt rect holding
  a string that typesets to 71.6 × 12pt. Using the rect as the value's geometry is a
  fabrication three times too wide, on a baseline nobody measured.
* **Rotation never meets widgets here.** The one rotated corpus document
  (`il_dol_day_labor_wage_notice_sample.pdf`, confirm, `/Rotate 90`) has no form fields.

## 3. The license (exactly one of the five): **the page's own print**

A widget value is what the page prints. Print the filled form on paper and the value is
on the paper, in that place, in that font. This is not an inference *about* the print
(as an OCR reading is) nor an adjacency guess — it is the document's own typed statement
of its glyphs and their positions. It is therefore licensed for **every** path, unlike
`core/ocr_words.py`'s words, which are confined to identity-closing paths precisely
because they are a guess at what the page prints.

**What is refused, explicitly: the widget's internal field NAME.** This carrier's names
are seductive — `EMPLOYEES NAME`, `AVERAGE HOURS PER WEEK`, `RATE OF PAY OR SALARY
HOURLY DAILY OR PIECE RATE` — and binding by them would light up all 7 fields today.
Refused, because an internal name is **not printed on the page**, and the house rule
admits nothing else. It is an authoring artefact: it varies per carrier with no
vocabulary anyone controls (`Text12` is as legal as `EMPLOYEES NAME`), and a name left
*stale* by a field duplicated and retitled in a form editor binds a real value to the
wrong field with an exact coordinate attached — the most convincing kind of wrong.
Only `/V` and geometry are read. Where the page prints no label the ordinary rules can
bind, the value is not read, and that abstention is the correct answer.

## 4. The rule (a new input source, not a new acceptance rule)

Each filled text widget's appearance stream is drawn onto a derived copy of its page at
the position PDF 32000-1 §12.5.5 puts it (`/BBox` through `/Matrix`, fitted to `/Rect`)
— the identical construction a viewer performs — and that derived page is read by
**`core.extract.read_words`**, the same function, through the same pdfminer, under the
same watermark filter. Fonts, advance widths, text matrices and baselines are resolved
by the engine that resolves them for printed text. **Nothing is estimated:** no
character width, no baseline, no size. The surviving words are merged into the page's
word stream and every existing rule applies unchanged, unaware the module exists.

New module `core/form_fields.py`; one insertion in `core/extract.py`'s page loop.
Flag `REALDOOR_FORM_FIELDS` (default on; `=0` restores byte-identity).

## 5. The hazards, named (each with the refusal that carries it)

* **Double-read on a flattened form** → `_drop_double_read`: a derived word is dropped
  when a printed word carries an equal string and their boxes intersect, or covers ≥50%
  of its box area. NOT `ocr_words.drop_overlapping` — that guard drops on *any*
  intersection, which is right for tight OCR quads and wrong for typeset em boxes.
  Applied here it silently deleted 5 of the 17 values, including both fields this
  iteration exists for (`$17.10 hourly` grazes the caption one line above it by 1.4pt,
  7% of its area). Adjacency is not duplication; the measured worst case is 0.075.
* **Checkbox/radio widgets** (`/FT /Btn`) → nothing is emitted. Their appearances are
  ZapfDingbats glyphs (a "4" that draws a checkmark) which would enter the word stream
  as a numeral. The filled carrier has 10, including `Every two weeks` = `/On` — a real
  `pay_frequency` statement this module cannot make. Filed, not smuggled.
* **Rotated pages** → nothing is emitted. A widget appearance may be rotated relative to
  its page (`/MK /R`) independently of `/Rotate`; no corpus document composes the two,
  so the composition is unfalsifiable here.
* **Unresolvable fonts** → the AcroForm `/DR` is merged *under* each appearance's own
  `/Resources`. Without it pdfminer resolves no font and returns correct text at **zero
  width**, silently. Observed during construction; it is why `/DR` is merged rather than
  trusted to be absent.
* **A document we cannot re-render** → every failure path returns empty, and the
  document is read exactly as it always was.

## 6. Firing predicate

`loop/falsify/it-012.py`. `fires(doc)` extracts each document twice in-process,
`REALDOOR_FORM_FIELDS=0` then `=1`, and fires iff the emitted field set differs.
`conflicts` joins every changed emission against that document's own truth, including
`expect_absent`. Swept over all 77 manifest documents plus the filled dev split and the
unsealed scenario sets. Sealed documents are never opened.

## 7. Predicted flips (exact)

* `filled::wa_dshs_14252_employment_verification_filled.pdf::person_name` → `Terrence Boyd`
* `scenarios::{S06,S10,S12,S15,S17,S28}` → `person_name` on each set's wa_dshs letter
* **Zero flips on all 77 manifest documents** (all 3 AcroForms there are blank).

No other field is predicted to move. The remaining 6 fields of the filled carrier are
predicted to keep abstaining: the form's printed captions (`AVERAGE HOURS PER WEEK`,
`RATE OF PAY OR SALARY…`) are not in the vocabulary — that is T21, and this iteration
must not reach it by reading widget names.
