## 2. The printed evidence (measured this iteration, `loop/worktmp/ocr_design/probe_t10*.py`)

**ou p1** (root cause corrected: THREE hours survivors, not the two the backlog
recorded): the anchored earnings run closes (column sums to printed 1,251.09; hours
column 3.57 + 68.43 + 8.00 = printed 80.00, which is what tells the factor columns
apart), but S3 sees three distinct hours values {3.57, 8.00, 68.43} and abstains. The
page itself names the row: the words `Hourly Regular` print at baseline 541.08, within
line tolerance of the anchored amount 854.06 (bl 541.45) whose hours factor is 68.43;
the other two anchored rows print `PAID LEAVE SCHEDULED` and `HOLIDAY LEAVE`. And the
page names the column: the header line prints `RATE | HOURS | EARNINGS` with `HOURS`
at x[224.0,239.0], dead over the hours column x[225.4,239.0], 6.2pt above its top row
— the nearest alphabetic run above the column, exactly `HOURS`, unqualified.

**osu p1**: the summary column [1666.94, 4327.06] sums to the printed total 5994.00 —
the very identity that emitted gross_pay in it-003 — and its last term is the net,
exactly the form `verified._bands` already accepts. What blocks it, measured: a
stray page token `1` (bl 616.4, a form field `PavId 1`) forms the degenerate product
`1666.94 x 1 = 1666.94`, which puts baseline 616.04 into `anchored_lines`, and
`_bands` refuses any run touching an "earnings" line. The band is being excluded by
the multiplicative identity. (The role-by-elimination alternative was measured and
died: no column run on the page closes to 1666.94, so "the member that closes over
its own detail" is not printed evidence here — the deduction detail does not sum in
the OCR stream.)

## 3. The license (exactly one of the five): **printed total closed by an arithmetic identity**

Both fields are emitted by the existing chain, completed, not by any new evidence
class. osu net: "the last term of a consecutive run summing to the accepted gross" is
the accepted rule of `verified._bands`/`verify_page` verbatim; this iteration only
stops a `x * 1 = x` coincidence from testifying that the band's line is an earnings
row — the multiplicative identity holds for every number beside a printed 1, so it
can anchor nothing and exclude nothing. ou hours: the emitted 68.43 is a factor of a
row product inside an anchored run whose hours column sums to a printed total (80.00)
— all identity — and the two anchor-only printed strings that pick the row and the
column (`REGULAR` on the row, a header that is exactly `HOURS` over the column,
the `shredded._ROW_LABELS` precedent) are exact-match after `normalize_label`, never
fuzzy (design hazard 6), and never become fields. L4 from the design's it-B list is
already subsumed: the frozen `product_tolerance` closed ou's truncated row in it-003,
so no tolerance is touched. Scope: both completions run **only on pages that carry
it-003-injected words** (`band_role=bool(injected words)` and the flag) — the text
path is not just bit-identical with the flag off, it is untouched with the flag on.

## 4. The rule (one mechanism per field, inside `verified.verify_page`)

`verify_page` gains a keyword `band_role: bool = False`; `extract_document` passes
`band_role = bool(injected) and _ocr_band_role_enabled()` (new flag
**`REALDOOR_OCR_BAND_ROLE`**, on by default, `=0` ⇒ byte-identical output). With
`band_role` true, exactly two lines of conduct change: (1) `anchored_lines` — the set
that excludes band runs — omits lines whose only claim to being an earnings row is a
product with a factor of exactly 1.0; (2) `regular_hours` candidates are no longer
every member of the hours column but the hours factors of anchored rows that print
the word `REGULAR` (word-level, same baseline tolerance the chain already uses),
admitted only when the nearest alphabetic run above the hours column that shares
x-extent with it is exactly `HOURS`, and only when the hours and rate value sets are
disjoint. Everything downstream — V1/V2/V3 vetoes, S3 one-distinct-survivor,
`certainty="low"`, the identity-naming note, `_with_ocr_provenance` — is unchanged
and still applies.

## 5. The hazards, named (each with the refusal that was measured to hold)

- **A wrong row named REGULAR** (the rule's own wrong value: overtime hours under a
  row that also prints REGULAR, e.g. "REGULAR OVERTIME"). Refused by S3: every
  REGULAR-named row contributes its hours factor, and two distinct survivors abstain.
  Measured: ou's prose `Regular Earnings` (bl 137.78) is on no anchored row and
  contributes nothing.
- **A qualified hours column emitting** (lcc prints `Hours or Units`, and its truth
  lists regular_hours in `expect_absent`). Refused twice: L5 exact-match `HOURS`
  fails on the qualified header by its own wording, and lcc's single-row table forms
  no anchored run at all (measured: no change on lcc).
- **The 111.75 band trap** (a real earnings run nominating its own member as net once
  the exclusion weakens). The exclusion weakens ONLY for lines whose every product is
  degenerate; a real earnings row carries a real rate x hours product (both factors
  != 1) and stays excluded. Residual: a page printing a true $1.00/hr rate row on an
  OCR page would lose its exclusion — the run must still sum exactly to the accepted
  gross and survive S3, and the sweep over all 77 finds no such page. Named, carried.
- **Role reversal** (a band printed net-above-deductions would put deductions last).
  Not new: this is the accepted `_bands` tail convention, unchanged; on osu the tail
  4327.06 is the printed `Netamount` row (measured), and only one closed run sums to
  the accepted gross, so one distinct survivor.
- **hi_ags multiplicity / misread totals**: no anchored run forms there (total
  misreads `'2.328.01'`), so neither completion can engage — measured, no change.
- **up_014 567-vs-700 trap, up_006, ca_dlse, pack**: measured, no change; dates and
  names remain structurally unreachable (`VERIFIABLE_FIELDS` untouched).
