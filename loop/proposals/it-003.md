# it-003 — T9 — embedded-image regions become Words for the identity paths, and nothing else

## 1. Target

Backlog **T9** (design it-A of `ocr-extension-design.md`), corpus `confirm`, docs
`ou_sample_check_stub.pdf` (hourly_rate `12.480800`, gross_pay `1,251.09`, net_pay
`750.14`) and `osu_sample_earnings_statement.pdf` (gross_pay `5994.00`) — all
`image_only`: the values exist only in the pixels of an embedded raster stub inside a
document whose prose text layer routes it down the text path.

## 2. The printed evidence (measured: design §A dossier + this iteration's native-scale probes)

ou p1 embeds one 1339×900 px stub at [72,95→562,489] pt (~197 dpi). At its own native
scale the engine reads every truth value verbatim and the page closes four independent
identities: the earnings column sums exactly to the printed TOTAL 1,251.09 (8 terms),
the band 225.24 + 275.71 + 750.14 = 1,251.09 exactly, the hours column 3.57 + 68.43 +
8.00 = printed 80.00, and the row products 12.4808 × {68.43, 8.00, 3.57} close at cent
grain. osu p1 embeds a 966×1320 px stub at 200 dpi printing the summary column
1666.94 + 4327.06 = 5994.00 with 5994.00 as its printed total. **Why the score is 0
today, verified**: the confirm harness calls `core.extract.extract_document` (text
path) for all 14; `ocr/ocr_extract.py` runs only when a caller routes to it, reads
page 1 only, and is fitted to the pack generator (design §1a–d). The zero is routing,
not readability.

## 3. The license (exactly one of the five): **printed total closed by an arithmetic identity**

The existing `verified.verify_page` / `shredded.recover` machinery — anchored column
sums, row products, total bands, S1/S3, V1–V3, the YTD separation — IS the license,
unchanged. This iteration adds no acceptance rule. It adds a *word source*: the
detections the OCR engine reads inside a page's own embedded-image regions, converted
to core `Word`s with engine geometry, and handed **only** to the identity-closing
paths. An OCR word may serve as an operand; it may be *emitted* only when the page's
own printed arithmetic closes over it. Grounding-wise nothing weakens: V1 still
requires the emitted string to have been read off the page (an unread value has no
token), and confidence gates nothing — closure is the gate (design §B.1.4).

Alternatives weighed and refused: extending `ocr/ocr_extract.py`'s label path to the
confirm docs — its geometry constants are pack-fitted by their own provenance comments
and label adjacency is the evidence class that shipped gross_pay=567.00-shaped errors;
re-OCRing full pages and diffing against the text layer — re-reads the page's own
printed numbers and can double them into column sums (manufactured closure).

## 4. The rule (one mechanism, no changes to verified/shredded logic)

New module `core/ocr_words.py`, called from `extract_document`'s page loop after
`read_words`, behind **`REALDOOR_OCR_WORDS`** (on by default; `=0` ⇒ the module is
never imported and output is byte-identical). Conduct, in order: (1) region discovery
by `page.images` XObject enumeration; intersecting placements are merged (union rect)
so overlapping instances can never be OCR'd twice into one stream; (2) render scale =
the region's own statement, max(src_px/placed_pt per axis), capped only by the
engine's existing input cap (`REALDOOR_OCR_MAX_SIDE`); crop-render of the region
alone, so memory is bounded by that cap; (3) `Word`s from engine geometry only:
per-character boxes (`return_word_box=True`) grouped at the engine's own space
characters; a detection whose char boxes do not reconcile with its text falls back to
one Word per detection — never proportional splitting; (4) overlap guard: an OCR word
whose box intersects any text-layer word's glyph box on that page is dropped; (5) the
injected words are appended to the page's word list **only** for the
`verified.verify_page` and `shredded.recover` calls — `extract_fields_from_page` (the
label/column/header-cell paths) never sees them, so no adjacency rule can emit an OCR
reading, structurally. Emitted values stay `certainty="low"` (both identity paths
already cap there); when an accepted token is an injected word, `extract_document`
appends a note naming the OCR provenance on top of the identity the paths already
name. `api/store.py` adds `core/ocr_words.py` to `ENGINE_SOURCES` so the extraction
cache key tracks the new engine source (loop README trap 1). `ocr/ocr_extract.py` is
not touched; the pack's rasterized path and its `certainty="high"` grandfathering stay
exactly as they are.

## 5. The hazards, named (each with the refusal that was measured to hold)

- **A digit flip surviving an identity by coincidence.** A misread operand breaks the
  sum unless the total misreads by the same delta; a product breaks unless a factor
  compensates (measured base rate: zero digit substitutions at ≥150 dpi across ~700
  digit-carrying detections; the one observed flip was at 96 dpi). The identity is the
  refusal; nothing rests on confidence.
- **The 567-net-vs-700-gross adjacency trap** (up_014, rotated+blurred scan): gross
  700.00 is unreadable and 567.00 (the net) sits under the GROSSPAY label. Refused
  twice: label adjacency cannot emit (structural gating), and no identity closes over
  567 as gross. Measured this iteration: up_014 emits nothing with injection on.
- **hi_ags page-2 multiplicity**: two stub instances printing different pay periods
  merge into one region (their placements intersect) and one word stream; no anchored
  run and no consistent band forms over the conflicting instances. Measured: hi_ags
  emits nothing on any of its 5 pages. Its p1 total misreads as `'2.328.01'`, which
  parse-refuses — the borderline fields stay abstentions for it-C, not wrong values.
- **Uniform decimal loss** (the one correlated misread family sums cannot refuse —
  every ratio still closes at ×100): the chain's anchor is a row *product*, which
  scales ×10⁴ while the amount scales ×10², so no anchored run forms. Measured on a
  synthetic decimal-stripped ou p1 (80 money tokens stripped): zero emissions.
- **Blank-form invention**: four confirm docs list every field `expect_absent`, and
  kcha carries two full-page ~117 dpi scans (Section 8 packet pages). Any emission is
  a scored WRONG. Measured: kcha's scans inject 190–305 words/page and emit nothing —
  the packet pages print no closable pay arithmetic. Same for md_labor/seattle/
  wa_dshs logo regions.
- **lcc's `expect_absent` rate/hours**: 31.00 × 10.51 = 325.81 closes on paper, but the
  single-row earnings table refuses the ≥2-member column-sum anchor, so nothing is
  emitted (measured) — exactly what truth demands there.
- **Scanned-text double-counting**: `ca_dlse_paystub_hourly.pdf` places a full-page
  image *under* its text layer; the overlap guard drops every OCR duplicate of a
  printed word, and the remaining fragments changed nothing (measured; its verifiable
  fields are also already answered, so `wanted` is empty there).
- **The YTD trap**: osu prints Current = YTD identically. Both columns close, both
  carry the same value; the emitted 5994.00 equals truth by identity, not by column
  choice — and ou's YTD family never entered a closed identity (measured note text).
- Dates and names **cannot** be emitted by this path at all: `VERIFIABLE_FIELDS` is
  the numeric four, so masked years/centuries stay uninvented structurally.

## 6. Firing predicate

`loop/falsify/it-003.py` — read-only; carries the candidate conduct verbatim (region
merge, native-scale crop render, engine-geometry word split, overlap guard) and runs
the full extractor twice in-process per document — injection patched in at exactly the
two call sites it will occupy vs. not — over all 77. Fires iff the emitted field set
differs; `conflicts` joins every changed field against that document's truth including
`expect_absent`. Sanity: must fire on both target documents and on **no pack document**.
